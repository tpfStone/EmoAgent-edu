# F9 候选文本错误类型汇总

这是一份基于 40 条 F9 候选回复的 AI 初标诊断稿。它用于指导 F3 生成器和 F4 评审器的 prompt 修复，不替代人工 ER/IP/EX 标注。

## 覆盖情况

- 来源：`docs/corpus/f9/baseline/f9_annotations_merged.csv`
- 分析行数：40
- 严重度：P1 = 36，P2 = 4，P0 = 0，OK = 0
- 修复归因：F3+F4 = 35，F3 = 5
- F4 误判：是 = 35，否 = 5

主要结论：F3 生成了大量带有明显 prompt 痕迹的回复，F4 基本没有惩罚这些模式。最强证据是：F4 对 40 条样本的 ER/IP 全部给了 2/2，而人工共识分有明显差异。更深层的原因不是单纯“F4 没写惩罚项”，而是 IP=2 的操作定义过松，容易把无依据动机推断误判成“听懂了”。

## 交叉审核后修订

- sample 16 从 `P1` 调整为 `P2`：二选一问法本身符合低压探索，问题在于问题前提替第三方解释了“别人可能有省力方法”。
- sample 22 不再归入 `边界/安全/不适龄`，改为 `关系决策风险/强化冲动决策`：它需要降分，但不等同于鼓励隐瞒、自伤风险等硬安全边界。
- F3 修复不再一刀切禁止正向承接，而是禁止“未充分承接情绪前，把痛苦直接转成优点”。
- F4 模板化惩罚改为看信息增量：如果模板句替代了对当前倾诉的具体回应，ER/IP 才应被限制。

## 高频坏模式

1. **品质化总结太多**
   - 典型句式：`说明你...`、`这本身...`、`挺难得`、`很有数`、`很勇敢`、`很有主见`
   - 问题：未充分承接情绪前，就把学生的痛苦、自责或愤怒包装成优点，像老师在评价表现。
   - 代表样本：3、4、9、10、17、22、35、38

2. **兜底安抚反复出现**
   - 典型句式：`先停在这儿也没关系`、`不用急`、`先缓一缓`、`先照顾好自己`、`没什么不对`
   - 问题：句子本身不一定错，但在大量场景里替代了真正的回应，读起来像 prompt 模板。
   - 代表样本：3、5、10、13、23、24、29、33、36、38

3. **形容词复述代替理解**
   - 典型句式：把原文换成 `委屈`、`憋屈`、`堵`、`累`、`沉`、`难受`、`不公平`
   - 问题：看似贴合，但没有解释当前场景里“为什么是这种难受”，信息增量很低。
   - 代表样本：3、6、10、15、24、29、33

4. **固定转折引导**
   - 典型句式：`不过你有没有注意到`、`不过我在想`、`换个角度看`、`我想轻轻递给你一个视角`
   - 问题：容易暴露“引导反思型 prompt”，像咨询师或老师在带步骤。
   - 代表样本：1、4、25、26、30、31、39、40

5. **替孩子或第三方下结论**
   - 替孩子下结论：`你其实在保护自己`、`你其实很清楚`、`你很想让他满意`
   - 替第三方下结论：`他们可能没顾上看你`、`班主任觉得你们扛得住`、`妈妈可能没意识到`
   - 问题：超出了“只根据文本”的原则，容易让学生觉得被解释、被教育。
   - 代表样本：1、7、13、19、27、28、31、34、39

## F3 生成器修复方向

- **承接先于重构。** 不得在未充分承接情绪前，把痛苦、自责、愤怒、不信任直接重构成“优点”“主见”“判断力”“在乎别人”。
- **轻量稳定感必须依附具体回应。** `先缓一缓`、`没关系`、`不用急` 可以偶尔用，但前面必须已经具体回应当前倾诉，不能作为默认结尾。
- **减少品质化夸奖。** 不要默认使用 `说明你...`、`这本身...`、`挺难得`、`很勇敢`、`很有主见`；除非文本依据非常具体。
- **避免替第三方读心。** 不解释父母、老师、同学、朋友“可能是怎么想的”，焦点放在学生感受到的影响和可控边界。
- **二选一问题可保留。** 好问题应该具体、低压、学生能直接回答；但问题前提不能替第三方解释动机，也不能替学生下人格或关系结论。
- **减少成人 coaching 语言。** 避免 `情绪地图`、`把这顿饭吃透`、`递给你一个视角` 这类成人咨询腔比喻。

## F4 评审器修复方向

- **收紧 IP=2 定义。** IP=2 只给有文本依据的隐含情绪/担忧命名；无依据动机推断、人格结论、因果解释不能算 IP=2。
- **模板化惩罚看信息增量。** 如果回复主要信息来自换词复述、泛化安抚或品质化夸奖，而不是当前倾诉的具体呼应，ER/IP 最高只能给 1。
- **第三方动机推测降 IP。** 无证据解释父母、老师、同学、朋友的动机时，轻则 IP 上限 1，严重替第三方开脱时 IP=0；只有事实编造或硬越界才进 `boundary_flag`。
- **关系决策风险单独处理。** 强化冲动断关系、羞辱、报复，或把不稳定关系决策夸成“有主见”，应降 ER/IP；不与真正安全边界混为一类。
- **EX 不是“只要提问就高分”。** 具体、低压、好回答的二选一问题可以给分；抽象、诱导、挑战式、成人 coaching 式问题不应给 EX=2。
- **F4 rationale 要显式报告模板风险。** 即使回复没有安全越界，反复出现 `说明你`、`换个角度`、`先缓一缓`、`挺难得` 也应影响分数或至少进入理由。

## 自动验收准入门槛

具体 run 的数字以 `docs/corpus/f9/validation/f9_validation_report.md` 为准。本 taxonomy 只定义进入正式人工 F9 前应满足的最低自动准入条件：

- 旧坏候选 F4 复评通过率至少 80%，即 10 条 golden 旧坏候选中至少 8 条符合预期。
- 旧坏候选 ER/IP 同时 2/2 的比例不超过 20%，避免继续把明显坏文本判成高质量。
- 新 40 条重跑样本中 ER=2 与 IP=2 的比例都不超过 80%，不能接近全满分饱和。
- F4 rationale 如果识别到模板化、第三方解释、事实补全、强行重构，分数必须实际降下来。
- F3 golden 与新 40 条重跑样本不得出现检测到的第三方事实/动机补全等 regression flags。
- 生成器不得 fallback。

如果这些门槛未通过，`validation/rerun/f9_rerun_blind_annotation.csv` 只能作为诊断包保留，不建议作为正式人工 F9 包发出。

## 验收口径

- 不追求“0 条 P1”。这批样本里 P1 高发说明底座模型有明显默认输出倾向，prompt 修复的现实目标是让重大坏模式明显下降。
- blocker：第三方读心/替权威开脱、事实补全/编造、强化冲动关系决策、成人 coaching 套话主导、F4 ER/IP 饱和满分。
- 可接受风险：少量形容词复述、偶发兜底安抚、在具体承接后出现的轻量稳定感、贴合语境的低压二选一问题。

## 人工复核建议

- 这版初标仍是诊断稿。复核重点不是逐条判断 AI 标得是否完美，而是决定哪些坏模式要变成 F3/F4 的硬规则。
- 建议优先复核样本：3、11、16、19、22、25、27、31、40；sample 15 作为防止过度惩罚的辅助对照。
