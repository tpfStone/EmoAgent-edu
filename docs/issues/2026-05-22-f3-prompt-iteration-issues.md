## 残留问题

### F3-RES-001：`c2` 仍会推测第三方心理或动机

严重度：major（prompt 质量问题；是否影响最终 `/chat` 结果还取决于 F4 是否选择该候选）

出现位置：

- 当前最好一轮：`f3-generator-sample-20260522-152533/generator_samples.md`
- 样本：`syn_0011`
- 候选：`c2`，引导反思型

现象：

`syn_0011` 是“最好的朋友在背后说坏话”的外放型同伴关系场景。最新一轮 `c2` 仍把新视角推向朋友的内部动机，例如暗示对方可能“嫉妒”或“心里虚”。这类内容不是孩子输入中给出的事实。

问题：

- 与当前 prompt 中“不要推测孩子没说出口的外部事实或他人动机”的约束冲突。
- 容易把孩子的愤怒从“我被背叛、我需要边界”带向“分析对方为什么这么做”。
- 对初中生场景可能显得像替对方找原因，削弱对孩子当下愤怒的承接。

期望：

`c2` 可以递出低压力新视角，但新视角应优先锚定孩子自己已经表达出的内容，例如：

- 我这么生气，是因为我真的在乎这段友谊。
- 现在先保护自己的感受和边界，比马上决定彻底断掉关系更重要。
- 等情绪没那么冲时，再决定要不要听对方解释或表达自己的底线。

不应替朋友、同学、家长或老师解释动机，也不应给出“对方其实害怕/嫉妒/心虚”等未被输入支持的判断。

#### 2026-05-23 小样本复验证据

复验 run：`small-sample-20260523-161056`

本轮 F3/F4 修复后，`syn_0012` 和 `syn_0032` 已不再出现内部提示外泄，也未复现“三科作业排序”事实编造；但 `/chat` 最终回复仍在 `syn_0012` 上推测家长动机：

> “也许不是她觉得你不够好，而是她太希望你好了，好到不知道该怎么表达，只能用这种最笨的办法。”

这说明第三方动机推测仍未被 F3/F4 稳定拦住。后续修复应继续收紧：遇到亲子/同伴冲突时，新视角只能落在孩子自己的感受、需要、边界、关系价值或可控表达上，不解释家长、老师、同学“其实为什么这么做”。

## 后续建议

- 下一轮若继续改 prompt，应专门强化“同伴/家庭冲突”分支：新视角只谈孩子自己的感受、需要、边界、关系价值和行动时机，不分析第三方心理。
- 可加入一条反例约束：当输入只描述他人做了伤人的事时，不要生成“对方可能其实……”结构。
- 改完后仍用同一组 5 条样本先跑 `/api/generator/generate`，确认 F3 本身稳定，再决定是否跑 `/chat` 检查 F4 选择。
- 本轮暂不处理 `LLM_MAX_TOKENS=1000` 与 spec 建议 `<=200` 的差异；这属于后续长度控制优化。

## 2026-05-23 MVP 45 条验收新增暴露问题

来源：`docs/acceptance/orchestrator-mvp/2026-05-21/runs/real-llm-20260522-215717/`

### F3-RES-002：内部提示外泄到最终回复

严重度：major

- 样本：`syn_0012`
- 现象：`c2`/最终回复末尾出现“如果孩子想继续，可以追问……”这类面向老师、评审或开发者的元说明。
- 修复口径：F3 最终输出只能包含直接对学生说的话；禁止内部提示、括号式教师提示、候选/策略/建议回复等元话术。F4 对此类候选应标记 `boundary_flag=true` 并排除。

### F3-RES-003：补充用户未说的事实

严重度：major

- 样本：`syn_0032`
- 现象：用户只表达作业写不完和老师各自留作业，`c2`/最终回复却写成“把三科的作业都列出来排了顺序”。
- 修复口径：F3 不得创造用户未说的数量、科目、排序、具体行动或第三方心理；引导反思型若需要补事实才能递出新视角，应退回为关于孩子感受的轻问题。F4 对明显事实编造候选应标记 `boundary_flag=true` 并排除。

### F3/F4-MVP-REMEDIATION：本轮修复边界

- 只做 MVP 缺陷闭环：F3 prompt 收紧 + F4 boundary 增强。
- 不新增后处理器，不改 `/chat`、F3、F4 API schema，不调整 EPITOME/CASEL 权重。
- “更加温柔”仅作为轻量负面约束：少评价、少说教、少替第三方解释，多承接孩子感受。
